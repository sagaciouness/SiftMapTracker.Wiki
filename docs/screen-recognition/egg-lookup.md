# 查蛋识别

查蛋识别从蛋详情中读取蛋名称、身高和体重，再用本地 S3 数据库验证结果。默认快捷键为 `Alt + E`。

## 首次配置

进入“采集校准 → 查蛋识别”，依次框选：

1. **蛋名称区域**：包含完整蛋名称和名称下方横线，右侧预留较长名称空间；不要框入类型标签和大面积背景。
2. **身高 + 体重区域**：一次框住两个数值和单位；不需要包含日期。

框选器左侧会显示示例图和当前步骤。拖动选择区域后按 `Enter` 确认，按 `Esc` 取消；也可以在校准中心分别重新框选其中一个区域。

完整说明见[采集校准](/screen-recognition/calibration)。

## 识别步骤

1. 在游戏中打开目标蛋的详情页。
2. 保持详情页可见，不要让其他窗口遮挡已框选区域。
3. 按 `Alt + E`，或点击“立即识别”。
4. 等待灵动岛展开并显示结果。
5. 在查蛋控制台查看 OCR 原文、解析后的身高/体重、匹配状态和耗时。

识别开始时，控制台会进入独立的“识别中”状态并清空上一轮 OCR 字段，避免把旧结果误认为本次结果。

## 结果状态

| 状态 | 含义 |
| --- | --- |
| 精确匹配 | 名称唯一命中，身高和体重均在可信范围内 |
| 容错匹配 | 名称经过受控 OCR 混淆修正后唯一命中，数值有效 |
| 多个候选 | OCR 信息不足以唯一判断；灵动岛会展开显示候选 |
| 数值越界 | 名称已唯一命中，但身高或体重不在该精灵范围内 |
| 未匹配 | 没有可靠名称候选 |
| 解析失败 | 蛋名称、身高或体重缺失，或单位无法解析 |

客户端不会仅凭身高和体重强行判断精灵。名称唯一但数值越界时，也不会改选另一个体型接近的精灵。

## 提高成功率

- 让游戏保持在校准时的分辨率、位置和缩放比例；
- 名称区域要覆盖完整长名称，但不要包含多余标签；
- 身高/体重区域要保留 `m`、`kg` 单位；
- 避免鼠标提示、弹幕或其他窗口遮挡；
- OCR 失败时先回到校准页使用“测试 OCR”检查两个区域。
