# 屏幕识别工作区

屏幕识别是与地图追踪并列的独立工作区，提供查蛋、童话事件监测、识别历史、精灵数据库和采集校准。它使用玩家框选的物理屏幕区域，不读取游戏进程、窗口句柄或内存。

![查蛋控制台](/screenshots/v0.9.3/screen-recognition-egg-console.png)

## 主要页面

| 页面 | 用途 |
| --- | --- |
| 查蛋控制台 | 按快捷键截取蛋名称和身高/体重区域，并匹配本地数据库 |
| 童话事件 | 监测“四叶草铅绘”，记录结果精灵、血脉与独立 80 次保底 |
| 识别记录 | 查看查蛋结果；时间按北京时间显示 |
| 精灵数据库 | 搜索可信精灵、体型范围和可孵状态 |
| 采集校准 | 为查蛋和童话事件分别框选屏幕区域 |
| 按键绑定 | 复用客户端统一的全局快捷键编辑页 |

## 使用前准备

1. 将游戏设置为稳定的分辨率、窗口位置和 Windows 缩放比例。
2. 打开“采集校准”，先完成所需功能的区域框选。
3. 在“按键绑定”确认查蛋和童话事件快捷键已成功注册。
4. 返回对应页面进行一次测试。

::: warning 显示器布局变化
保存的是物理屏幕坐标。更换显示器、修改分辨率或缩放、改变主屏幕或显示器排列后，客户端会将配置标记为失效，必须重新框选。
:::

## 两种识别任务

### 查蛋

查蛋是单次操作。按下 `Alt + E` 后只截取两个已校准区域，OCR 完成后立即停止本次采集。它可以在地图工作区直接触发，不需要启动地图追踪服务。

详见[查蛋识别](/screen-recognition/egg-lookup)。

### 童话事件

童话事件是显式启停的监测任务。监测时只读取名称小区域；确认事件进入结算后，才读取结果横幅。开始监测会停止地图追踪，暂停后不会自动恢复地图追踪。

详见[童话事件与保底](/screen-recognition/fairy-events)。

## 任务互斥

查蛋和童话事件共享一个 OCR 子进程，但不会并发执行：

- 童话事件监测期间按查蛋快捷键，会提示先暂停监测；
- 单次查蛋正在处理时，不能同时开始童话事件；
- 暂停时不截图、不执行 OCR；
- OCR 服务异常只影响屏幕识别，不应关闭客户端或地图服务。
