# 采集校准

查蛋和童话事件共用校准中心，但分别保存自己的物理屏幕区域。

![采集校准中心](/screenshots/v0.9.3/screen-recognition-calibration.png)

## 查蛋区域

### 蛋名称区域

- 覆盖完整蛋名称；
- 保留名称下方横线有助于确认位置；
- 向右预留较长名称空间；
- 不要包含详情页类型标签或大面积背景。

### 身高 + 体重区域

- 一次覆盖两个数值及其单位；
- 保留 `m` 与 `kg`；
- 不需要包含日期或其他说明文字。

## 童话事件区域

### 右上角目标名称区域

覆盖完整精灵名称。等级和血量可以在区域内，但区域应保持紧凑。

### 结果横幅区域

覆盖完整血脉句子，并确保“发现异色精灵！”提示不会落在区域外。

## 框选器操作

1. 点击“依次框选两个区域”。
2. 按左侧示例和步骤提示拖出第一个区域。
3. 按 `Enter` 确认并进入第二步。
4. 完成第二个区域后再次确认。
5. 回到校准中心检查显示器、坐标和像素尺寸。

可单独使用“仅框选蛋名称”“仅框选身高/体重”，或重新执行童话事件的双区域校准。

::: tip
框选器使用普通桌面截图作为背景。它不会读取游戏窗口信息，也不会要求游戏进程句柄。
:::

## 显示器与 DPI

校准数据包含保存时的显示器设备名、物理像素边界和两个区域。以下变化会使配置失效：

- 分辨率变化；
- Windows 缩放比例变化；
- 主显示器变化；
- 显示器排列或虚拟桌面坐标变化；
- 原显示器被断开。

校准页与实际识别都会实时核对当前 `Screen.AllScreens` 布局。出现“原显示器不可用”或“屏幕布局已变化”时，请重新框选，不要继续使用旧坐标。

## 测试与诊断

- “测试 OCR”会按当前查蛋区域执行一次识别；
- 默认不保存任何截图；
- 启用诊断模式后，只保存两个裁剪 ROI，不保存完整屏幕；
- “清理诊断图”只删除诊断 ROI，不影响校准和历史。
