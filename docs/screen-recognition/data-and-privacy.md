# 数据库、历史与隐私

屏幕识别的数据匹配和历史记录都在本机完成，不需要联网查询精灵信息。

## 精灵数据库

![精灵数据库](/screenshots/v0.9.3/screen-recognition-database.png)

当前 S3 数据库包含：

- **990 条**可信基础记录；
- **232 条**存在可孵蛋体型；
- **758 条**标记为“此体型蛋无法孵出”；
- 童话事件按同名身份合并为 **673 个**可搜索目标；
- 不可信或名称污染的数据保持隔离，不参与运行时匹配。

数据库页支持“全部记录 / 可孵出 / 此体型蛋无法孵出”筛选，并显示基础体型范围和可孵蛋范围。无法孵出的记录仍可作为童话事件目标，因为童话事件并不依赖蛋体型。

## 查蛋匹配边界

匹配使用可信名称、别名和身高/体重范围：

1. 先规范化 OCR 名称；
2. 精确命中名称后验证数值范围；
3. 精确失败时只应用受控 OCR 混淆修正；
4. 多个候选不会被擅自归为唯一结果；
5. 标题或数值缺失时不会仅凭体型猜测精灵。

## 本机记录

查蛋记录和童话事件审计写入：

```text
workspace/screen_recognition/
├── history.jsonl
└── fairy_events/
    ├── calibration.json
    ├── progress.json
    └── events.jsonl
```

- 时间以 UTC 保存，在界面统一转换为北京时间；
- 查蛋历史支持搜索、状态筛选和清空；
- 童话事件记录包含结果、血脉、计数变化和人工调整；
- 历史采用有限尾部读取，文件增长不会要求每次把全部记录载入内存；
- 设置和进度使用原子替换，降低异常退出造成半写入的概率。

## 截图与进程边界

屏幕识别不会：

- 读取游戏内存；
- 注入游戏进程；
- 枚举或绑定游戏窗口句柄；
- 持续录制全屏；
- 默认保存截图；
- 把 OCR 结果上传到在线数据库。

查蛋只在快捷键触发后截取两个区域。童话监测只截取名称小区域，结算时才额外截取横幅。诊断模式也仅能保存这两个 ROI。
