# 屏幕识别故障排查

## 页面显示“需要校准”

进入“采集校准”，完成当前功能要求的两个区域。仅有一个区域时配置不会生效。

## 显示“原显示器不可用”或“屏幕布局已变化”

显示器设备、分辨率、缩放或排列与保存时不同。将游戏放到目标显示器后重新框选。不要手动修改坐标，也不要复制其他玩家的校准文件。

## OCR 服务不可用

1. 确认发布包完整解压；
2. 检查屏幕识别服务文件是否被安全软件隔离；
3. 退出并重新打开客户端；
4. 仍失败时保留启动错误日志和屏幕识别服务日志。

OCR 子进程异常不应关闭客户端或地图追踪服务。

## 查蛋仍显示旧结果

新版本在识别开始时会显示“正在识别”，并清空蛋名称、身高和体重字段。若仍看到旧数据，确认客户端已更新到包含屏幕识别工作区的最新版本。

## 查蛋解析失败

- 确认游戏蛋详情位于校准时的位置；
- 名称区域应包含完整名称，长名称右侧需要留空间；
- 身高/体重区域要同时包含数值和 `m`、`kg`；
- 避免其他窗口、鼠标提示或动态内容遮挡；
- 使用校准页“测试 OCR”检查原始文字。

## 童话监测无法开始

依次检查：

1. 已选择目标精灵；
2. 名称区和结果横幅区均有效；
3. 当前次数小于 80；
4. 数据库更新后已重新确认目标；
5. 没有正在运行的查蛋请求；
6. 没有未处理的待复核事件。

## 童话事件漏记或误记

- 名称区域保持紧凑，并覆盖完整精灵名称；
- 横幅区域覆盖整句血脉文字和异色提示；
- 游戏窗口不要在监测过程中移动；
- 不要在分辨率或缩放变化后继续使用旧校准；
- 发生待复核时先查看 OCR 原文，再选择计入或作废。

## 快捷键没有响应

打开“按键绑定”检查注册状态。`Alt + E` 和 `Alt + T` 可能被其他程序占用，可改为不冲突的组合键后保存。

童话事件正在监测时，查蛋快捷键只会提示先暂停监测，这是任务互斥的正常行为。

## 性能异常

正常情况下只有一个 OCR 子进程和一个童话监测循环。暂停童话事件后不应继续截图或 OCR。若 CPU、线程或内存持续增长，请记录：

- 客户端版本；
- OCR 服务状态；
- 当前显示器分辨率与缩放；
- 是否开启诊断模式；
- 是否正在童话监测；
- 可稳定复现的操作步骤。
