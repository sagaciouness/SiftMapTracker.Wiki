# 童话事件与保底

童话事件模块监测“四叶草铅绘”事件，识别结算精灵与血脉，并按目标精灵独立维护固定 80 次保底。

![童话事件页面](/screenshots/v0.9.3/screen-recognition-fairy-events.png)

## 开始监测

1. 在“目标精灵”中按名称、别名或 ID 搜索。
2. 选中目标并点击“选择为目标”。
3. 打开“采集校准 → 童话事件”，框选名称区和结果横幅区。
4. 返回童话事件页，点击“开始监测”或按 `Alt + T`。

成功开始后，地图追踪会停止并复位地图 HUD。暂停童话监测后，地图追踪不会自动重新启动。

## 两个采集区域

- **右上角目标名称区域**：覆盖完整名称；等级、血量可以保留，但不要包含大面积背景。
- **结果横幅区域**：覆盖完整血脉句子，并确保“发现异色精灵！”提示位于区域内。

监测期间只持续读取名称小区域。只有事件进入结果阶段后，服务才会同时读取名称区和横幅区。

## 状态流程

```text
童话待命
→ 正在监测
→ 正在确认四叶草铅绘
→ 童话事件已激活
→ 正在读取事件结果
→ 已完成 / 待复核 / 错误
```

“四叶草铅绘”需要两次独立、可靠的 OCR 确认。事件结果名称也需要稳定确认，以降低动画、遮挡和短暂误识别造成的重复计数。

## 保底规则

- 每个目标精灵分别保存当前轮次和 `0–80` 次数；
- 可靠完成一次事件后次数 `+1`；
- 命中目标时，本次先计入，再归档本轮并创建次数为 0 的下一轮；
- 切换目标前会暂停监测；重新选回旧目标时恢复该目标的进度；
- 非目标结果使次数达到 80 时会显示“保底数据异常”，不会伪造目标命中；
- `+1`、`−1` 和“重置”均写入审计记录，重置需要确认。

## 血脉与异色终止

当前识别以下血脉：

| 血脉 | 识别内容 |
| --- | --- |
| 奇异 | “染上了几笔奇异的颜色” |
| 污染 | “染上了几笔污染的颜色” |
| 混乱 | “染上了几笔混乱的颜色” |
| 异色 | “染上了几笔异色的颜色” |
| 未识别 | 结果精灵可靠，但横幅未能读出血脉 |

当横幅以足够置信度识别到 **“发现异色精灵！”** 时，该提示具有更高优先级：

1. 当前事件只计数一次；
2. 血脉记为“异色”；
3. 归档本轮最终次数；
4. 创建下一轮并暂停监测；
5. 同帧能识别结果名称时保存精灵，否则记录“未知异色”。

普通的“染上了几笔异色的颜色”只表示血脉，不会单独触发提前终止。

## 人工复核

名称持续抖动、结果无法唯一确认或恢复中的事件缺少连续画面证据时，会进入“待复核”。此时可：

- 选择实际结果并计入；
- 结果未知但计入一次；
- 作废事件。

处理前不能继续监测，也不会自动增加保底次数。所有决定都会保留在事件审计中。
