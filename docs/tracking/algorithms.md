# 算法与参数

## 算法怎么选

| 算法 | 推荐场景 | 主要限制 |
| --- | --- | --- |
| SIFT | 日常使用、地图缩放或旋转存在小幅变化 | 低纹理区域仍可能无法定位，计算量较高 |
| TM | 小地图比例固定、画面清晰、目标区域较唯一 | 对比例变化、重复纹理和遮挡敏感 |
| ORB | 特定设备上的轻量尝试 | 设备差异较大，明显卡顿时应切回 SIFT |

## 刷新间隔

“刷新间隔”主要控制服务状态输出频率，并不等同于 SIFT 每秒识别次数。默认 `50 ms` 已兼顾响应和客户端解码负载。

::: warning 不建议设置为 35 ms
较低间隔会增加 JSON、预览图和 UI 调度压力，却不会按比例提高定位能力。旧版本中，只有坐标的新帧还可能覆盖携带小地图图片的帧，使“力大飞砖”指针看起来停止更新。出现该问题时恢复 `50–60 ms` 并重启追踪。
:::

## SIFT 参数

- **Match Ratio**：越低通常越严格，误匹配减少，但更容易识别不到。
- **最小匹配点数**：提高后更稳，但低纹理区域更难通过。
- **RANSAC 阈值**：控制几何一致性的容差。
- **CLAHE 增强**：改善亮度不均和阴影场景，不代表一定提升所有画面。

不清楚参数含义时使用“恢复默认”，一次只修改一个参数并在同一地点对比。
