# 数据格式

## 路线 JSON

当前路线格式以 `nodes` 和 `edges` 表示图结构，同时兼容旧版 `points`。路线至少包含稳定名称和地图坐标。读取旧文件后不应仅因查看而覆盖其内部名称。

## 资源 JSON

资源文件保存资源集名称、物品类型和点位。资源编辑器负责生成对应 cache PNG 与 manifest；地图标记页只引用 cache PNG，不重新解析点位。

## 测绘项目

测绘文档使用统一图层栈：

- 路线层：来源引用与节点/边快照；
- 资源层：来源 cache PNG 引用与项目资产副本；
- 标记层：文字、点位、画笔和基础形状。

系统图层内容只读，但图层本身可以排序、改名、隐藏、锁定和调整透明度。

## 运行时配置

- `settings_override.json`：用户参数和截图区域；
- `control_state.json`：客户端到服务的实时控制状态；
- `client_state.json`：客户端页面、路线和用户界面状态。

写入上述文件时必须采用原子替换；读取失败时保留上一份有效状态，不能因短暂半写入清空导航。
