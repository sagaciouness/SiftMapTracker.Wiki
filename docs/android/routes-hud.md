# 手机端路线与 HUD

## 定位算法

手机端 v0.9.0 提供：

- **仅模板匹配**：默认选项，适合比例稳定且画面清晰的小地图。
- **仅 SIFT**：对局部旋转和特征地标更有适应性，但通常计算量和耗电更高。

切换算法后，如果追踪正在运行，软件会自动重新开始定位。

## 选择和导入路线

主页路线下拉框同时显示内置路线与玩家导入路线。选择后 HUD 使用该路线，不会在运行中自动切换到其他路线。

导入方法：

1. 点击“前往路径站获取路线”下载 JSON。
2. 返回软件，点击“导入路线 JSON”并选择文件；也可以在文件管理器中使用“打开方式”交给本软件。
3. 导入成功后软件会自动选中该路线。

手机端可以使用当前路线分享站提供的路线文件。卸载应用会清除已导入路线，重要文件应另行保留。

## 路线总览

“查看路径总览”会显示当前路线的完整地图、节点和连线，支持单指拖动与双指缩放。

::: info 当前限制
Android 路线总览只展示路线全貌，不显示玩家实时位置。实时玩家位置只在悬浮 HUD 中显示。
:::

## HUD 设置

| 设置 | 范围或默认值 |
| --- | --- |
| 灵动岛大小 | 紧凑 / 标准 / 中等 / 最大，默认标准 |
| 透明度 | 35%–100%，默认 45% |
| 地图缩放 | 75%–300%，默认 75% |
| 截图预览 | 默认关闭 |
| 详细状态文字 | 默认开启 |
| 玩家地图视窗 | 默认开启 |

“恢复灵动岛默认”会恢复上述默认值。“重置 HUD 位置”只恢复悬浮窗位置，不会清除小地图框选和路线。

## 运行建议

- 日常使用可关闭截图预览，减少持续重绘。
- 发热或掉帧时先使用默认地图缩放和标准尺寸，再比较 SIFT 与 TM。
- 玩家指针使用原始定位坐标，路线用于显示和进度提示，不会把玩家位置吸附到路线。
