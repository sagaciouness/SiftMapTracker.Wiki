# Android 故障排查

## HUD 是黑屏或冻结画面

最常见原因是录屏授权时游戏没有已经显示，或选择了“单个应用”。

1. 点击“停止采集”。
2. 先打开游戏并保持小地图可见。
3. 用小窗或分屏打开本软件。
4. 点击“重新授权采集”。
5. 选择“整个屏幕 / Entire screen”，不要选择单个应用。

如果系统始终把游戏画面标记为受保护内容，系统录屏也无法采到游戏，该设备或当前游戏版本可能无法使用 Android 客户端。

## 点击框选没有游戏截图

- 必须先完成屏幕采集授权，并返回游戏等待产生真实帧。
- 状态未显示采集帧就绪时，不要立即框选。
- 如果软件小窗会自动全屏，改用通知栏中的“框选小地图”。

## 无法开启 HUD

- 检查悬浮窗权限是否仍然开启。
- 必须先保存小地图区域。
- Android 13 以上允许通知，便于确认前台采集状态并使用通知栏控制按钮。
- 部分省电策略会停止后台服务，可将本软件从强制省电或后台清理中排除。

## 一直显示搜索中

- 重新框选，确认没有包含小地图外 UI。
- 前往道路、建筑等纹理明显区域重新定位。
- 检查游戏小地图比例和朝向是否稳定。
- 分别测试“仅模板匹配”和“仅 SIFT”。
- 海面、雪地和大面积草地等低纹理区域可能无法稳定定位，这不是提高刷新频率就能解决的问题。

## 路线导入失败

- 文件必须是有效 JSON，并包含非空 `nodes` 或 `points`。
- 文件不得超过 4 MiB、10,000 个点或 20,000 条边。
- 某些文件管理器会给 JSON 错误的 MIME 类型，可使用软件内“导入路线 JSON”重新选择。

## 发热、耗电或掉帧

- 关闭“显示截图预览”。
- HUD 使用“标准”或“紧凑”，地图缩放恢复 75%。
- 比较 TM 与 SIFT，选择当前设备更稳定的一项。
- 停止使用时点击“停止采集”，仅关闭主界面不会等同于停止前台采集服务。

## 更新检查失败

确认网络可访问更新地址，然后在主页点击“检查更新”。下载入口可能随版本变化，以弹窗当时提供的链接为准。
