# 版本记录

正文只描述最新稳定版。本页保留各版本的主要变化，便于升级和排查版本差异。

::: info 当前算法
当前版本只提供 SIFT 与模板匹配（TM）。ORB 仅在部分历史版本中出现，现已移除。
:::

## 手机端 v0.9.0

- 首个与 S3 地图同步的手机端稳定版本。
- 支持手机屏幕采集、小地图框选、地图定位、路线导入和悬浮 HUD。
- 定位算法可选择 SIFT 或模板匹配。
- 手机端为精简版本，不包含电脑端的路线编辑、资源编辑、地图标记和图片导出。

## Windows v0.9.3

- 修复追踪服务启动被系统策略阻止时的客户端恢复流程。
- 提高小地图框选的最小尺寸，拒绝明显过小的区域。
- 延续 v0.9.x 的多显示器物理像素区域校验、资源缓存性能优化和 S3 赛季资源。

## Windows v0.9.1

- 修复 125% / 150% 缩放和多显示器环境下合法区域被误判越界。
- 截图服务改用物理像素虚拟桌面边界，支持负坐标。
- 首次启动追踪要求在当前电脑重新框选小地图。
- 运行中重新框选后自动重启服务。
- 补充玩家包设置默认值和 S3 赛季资源目录。

## Windows v0.9.0

### S3 赛季适配

- 更新 S3 地图、地图资源和定位风险图。
- 更新赛季野生异色识别模型及相关物品图标。

### 核心功能

- 新增“图片导出”工作台，支持点位、文字、画笔、矩形、圆形、箭头等标记，可自由绘制并导出图片。
- 新增“定位风险图”拓展，静态展示 SIFT / TM 理论定位风险区域，方便查看稳定追踪区域。

### 导航与 HUD

- 修复服务异常退出后 HUD 与导航状态不同步的问题，服务停止时会同步关闭路线导航并清理实时状态。
- 修复路线文件损坏、半写入或结构异常时导航被清空的问题；保留上一条有效路线和当前进度，待文件恢复后自动重新加载。
- 优化服务帧处理，优先显示最新画面、隔离单帧异常并恢复后续刷新，减少 HUD、预览和异色提示卡顿或停止更新。
- 修复自定义路线名称可能被文件名覆盖的问题，并记住上次使用的路线，重启后可自动恢复。
- 路线忽略改为精确文件名匹配，并在 5 分钟后自动恢复推荐。

### 性能优化

- 资源 JSON 改为后台读取和解析，快速切换时自动丢弃过期任务。
- 保持 8192×8192 地图原始画质，同时共享地图位图，减少重复解码与内存占用。
- 资源缓存改为后台生成，并增加缓存版本、哈希和来源校验，减少阻塞和误用过期缓存。
- 优化资源列表虚拟化及隐藏页面刷新逻辑，编辑器与导航切换更流畅。
- 资源图层改为独立 PNG 快照，不再依赖可能被替换的缓存文件。

### 界面与体验

- 重构主界面仪表盘、功能拓展页及编辑器布局；功能拓展卡片改为双列布局。
- 优化侧栏展开、收起动画、文字对齐和快捷功能区。
- 新增云端公告横幅。
- 资源库调整为本地资源库，异常图标地址不再中断资源加载。

### 稳定性与安全

- 修复资源缓存读写竞争可能导致程序退出的问题。
- 修复测绘项目未保存时打开其他项目或退出导致内容丢失的问题；保存失败时保留编辑状态。
- 设置、快捷键、客户端状态、路线、资源和导出 PNG 改为原子写入，降低半写入、缓存错乱和加载失败概率。
- 修复“恢复默认设置”会丢失小地图框选区域和地图校准原点的问题。
- 增强截图区域、跨屏 DPI、更新链接、文件与目录打开的校验和异常保护。
- 优化模型评估与 ONNX 校验工具的权重加载安全性。

## Windows v0.8.5

- 修复 SIFT / Template Match 在战斗、传送、遮挡等画面剧变时定位乱跳的问题。
- 重定位确认期间灵动岛保持上一已知位置，并提示“搜索中 / 重定位中”。
- 优化 Template Match 定位逻辑，新增多尺度自动搜索机制，改善低纹理区表现。
- 修复“恢复默认”清空地图校准参数并导致路线整体偏移的问题。
- 重做灵动岛地图渲染，改善底图与玩家指针移动的流畅度。
- 修复移动时地图边缘露出黑色背景的问题。
- 修复玩家指针被路线吸附点锁定、不显示真实位置的问题。
- 提高资源底图缩放清晰度，并优化类黄色地形下的指针描边。

## Windows v0.8.1

- 新增野生炫彩检测，使用轻量 ONNX 自训练模型。
- SIFT 启动内存峰值降低约 93%，启动耗时降低约 58%。
- 导航总览支持自动置顶且尽量不抢占游戏键盘焦点。
- 加粗放大导航总览中的路线、节点、玩家点和当前目标。
- 玩家位置、距离、锁定状态和底部状态栏改为实时刷新。
- 修复导航总览不显示玩家红点及“居中玩家”只能居中路线的问题。
- 优化客户端拖拽、导航总览加载、HUD 小地图与预览图刷新流程。
- 玩家红点改为独立动态图层，路线刷新不再影响玩家位置显示。
- 总览采用玩家状态高频更新、路线低频重绘策略。
- 修复主界面“实时轨迹视图”在部分机器上抖动的问题。
- 启动失败时写入本地日志，便于定位异常。
- 增加小地图区域快速重新框选快捷键。
- 修复候选路线清空后旧路线残留的问题。
- 客户端最小化后自动降低刷新频率。

## Windows v0.7.1

- 新增单路径 / 多路径导航模式，多路径最多选择 7 条候选路线，主路线固定参与。
- 对草地区域进行特征补强，加入游戏内标点作为稳定参照。
- 候选路线支持不同颜色，并同步到路线预览、导航总览和 HUD。
- 多路径切换改为手动确认，HUD 可确认或忽略更匹配的路线。
- 新增路线切换反馈灵动岛，分别显示切换中、成功、超时和失败。
- 新增“在此位置重新导航”快捷键 `Ctrl + Shift + F10`。
- 新增 HUD 视觉平滑开关。
- 优化候选路线异步加载和导航总览刷新策略。
- HUD 加入 mipmap 渲染缓存，移动时优先使用低分辨率图层裁剪。
- 缓存路线段并复用每帧枚举结果，减少对象分配、GC 和重复计算。
- 多路径建议加入候选缓存，玩家移动不足一定距离时复用结果。
- 玩家坐标变化时只刷新视口和玩家点，不再重建全部 HUD 文本及动画。
- 平滑居中刷新加入最小间隔，地图面板隐藏时停止刷新。
- 缓存并冻结固定颜色画刷，减少 UI 线程分配。
- 服务端对预览图和小地图图片进行产出节流，降低 Base64 编码、JSON 传输和客户端解析压力。
- 修复候选路线关闭后 HUD 残留、路线切换写入失败后状态不一致及路线元数据缓存刷新不及时的问题。

## Windows v0.6.7

- 新增“导航总览”独立窗口，可查看大地图路线、当前位置、当前目标与路线进度。
- 修复深色模式下侧栏、导航栏等区域的可读性问题。
- 新增自定义快捷键能力。
- 灵动岛传送点提示与导航总览联动。
- 重构 UI 并修复若干界面问题。

## Windows v0.6.0

- 小地图框选改为两阶段流程：左键粗选后进入可调整界面，再确认保存。
- 修复框选小地图后可能覆盖设置并将追踪算法重置为 SIFT 的问题。
- 新增宝箱、矿物、战斗、收集、植物资源点，数据来自 Rocopath，作者随我所想。
- 为当时提供的 ORB 加入动态框选尺寸、短时丢失惯性预测、局部搜索和全局恢复调整。
- 资源编辑、保存和结束绘制时自动重建缓存图，并同步刷新路线编辑与导航显示。
- 切换、保存和生成资源缓存时增加进度提示。
- 页面切换时仅在资源变更或缓存缺失时自动重建缓存。
- 路线编辑页与灵动岛导航层优先读取资源缓存图，减少逐点渲染卡顿。
- 重构软件 UI 布局，并在路线编辑页增加路线分享站入口。

## Windows v0.5.0

- 引入 ORB 算法以尝试改善草地区域抖动并降低 CPU 占用；该算法已在后续版本移除。
- 设置页按追踪算法切换对应参数页面。
- 小地图输入链统一为圆形有效区域。
- 路线编辑器支持节点类型、分支起点、线工具悬停预高亮、单点拖动及传送点浅蓝虚线。
- 路线文件优先读取 `nodes + edges`，继续兼容旧版 `points` 并转换为内部边结构。
- 资源点编辑器支持按分类和物品图标新增、拖动、删除、撤销、保存与导出。
- 资源点层从逐点控件改为单层集中绘制，改善大量资源点时的编辑流畅度。

## Windows v0.4.0

- 新增“游戏指针（力大砖飞版）”，可从小地图中提取游戏指针显示到灵动岛。
- 箭头改为白色主体、灰色描边，并放大以提高路线辨识度。
